# ROADMAP.md – Статус MVP и план масштабирования

Текущий статус разработки HostGuard AI и дорожная карта развития от MVP до v2.1.

## 1. Текущий статус MVP

| Компонент                     | Статус           | Комментарий                                           |
| ----------------------------- | ---------------- | ----------------------------------------------------- |
| Мониторинг Avito              | ✅ PoC (92%)     | Tavily API, стабильный парсинг                        |
| Мониторинг Островок.ру        | ✅ PoC (84%)     | Требуется Browser Rendering                           |
| Мониторинг Яндекс Путешествия | ⚠️ PoC (78%)     | Смешанный подход, доработка после MVP                 |
| AI-анализ (DeepSeek V3.2)     | 🔧 В разработке  | Системный промпт готов, требуется интеграция          |
| Telegram-уведомления          | 🔧 В разработке  | Инлайн-кнопки «Скопировать», «Одобрить», «Доработать» |
| Mobile-first дашборд (Vue 3)  | 🔧 В разработке  | Базовый каркас, карточка инцидента                    |
| Дедупликация (Redis)          | 🔧 В разработке  | Ключи `review:{platform}:{id}`, TTL 90 дней           |
| HITL-подтверждение            | ✅ Спроектирован | Архитектура готова, реализация в процессе             |

### Целевые метрики MVP

| Метрика                | Цель      | Текущий статус                 |
| ---------------------- | --------- | ------------------------------ |
| Scrape Success Rate    | ≥ 95%     | 78–92% (зависит от площадки)   |
| Время реакции на отзыв | < 3 часов | – (не замерялось)              |
| Hallucination Rate     | < 5%      | – (ожидается после Eval Suite) |
| Retention W1           | ≥ 50%     | – (после запуска)              |

## 2. Этапы разработки

MVP (v1.0) → Стабилизация → v1.1 → v1.2 → v1.3 → v2.0 → v2.1

### MVP (v1.0) – Апрель 2026

**Платформы:** Avito, Островок.ру, Яндекс Путешествия

**Функциональность:**

- Мониторинг 3 площадок с интервалом 1–4 часа
- AI-анализ тональности и нарушений
- Генерация публичного ответа и черновика апелляции
- Telegram-уведомления с инлайн-кнопками
- Mobile-first дашборд на Vue 3
- Дедупликация отзывов через Redis

**Ограничения:**

- Только 3 площадки
- Только Telegram-уведомления
- Нет интеграции с PMS
- Только ручная публикация (HITL)
- Поддержка русского языка

---

### v1.1 – Интеграция с PMS

**Срок:** После стабилизации MVP (2–3 месяца)

**Добавляется:**

- Интеграция с Bnovo, Realty Calendar, TravelLine
- Проверка факта проживания гостя через PMS
- Автоматическое определение: гость реальный или фейковый отзыв

**Влияние на метрики:**

- Повышение точности апелляций (факт отсутствия бронирования = железный аргумент)
- Рост Success Rate апелляций до 45–50%

---

### v1.2 – Расширение списка площадок

**Срок:** 4–6 месяцев после MVP

**Добавляется:**

- Суточно.ру
- 2GIS
- Яндекс Карты
- Google Maps

**Технические изменения:**

- Новые парсеры для каждой площадки
- Расширение Prompt Registry правилами новых площадок
- Адаптация UI дашборда под новые источники

---

### v1.3 – Автоматическая публикация

**Срок:** 6–8 месяцев после MVP

**Добавляется:**

- Публикация одобренных ответов через официальные API площадок
- Статус-трекинг апелляций (отправлено → на рассмотрении → удовлетворено/отклонено)

**Важно:** HITL сохраняется на этапе одобрения. Автоматизируется только техническая публикация после подтверждения владельцем.

---

### v2.0 – Мультиязычность

**Срок:** 9–12 месяцев после MVP

**Добавляется:**

- Поддержка английского и китайского языков
- Мультиязычные промпты в Prompt Registry
- Локализация дашборда

**Целевая аудитория:** Отельеры, работающие с иностранными гостями

---

### v2.1 – Собственный fine-tuned LLM

**Срок:** 12–18 месяцев после MVP

**Добавляется:**

- Тонкая настройка модели на основе накопленного Golden Set
- Снижение стоимости инференса в 3–5 раз
- Полный контроль над моделью (безопасность, скорость, предсказуемость)

**Технические изменения:**

- Переход с DeepSeek V3.2 на собственный хостинг модели
- Интеграция с Cloudflare Workers AI или выделенный inference-сервер
- Обновление Eval Suite под новую модель

## 3. Приоритеты на ближайшие спринты

### Спринт 1 (текущий)

- [ ] Завершить интеграцию DeepSeek V3.2 через AI Gateway
- [ ] Реализовать Telegram Bot с инлайн-кнопками
- [ ] Базовый каркас дашборда (список объектов, статус мониторинга)

- [ ] Дедупликация отзывов через Redis
- [ ] Карточка инцидента в дашборде

## 4. Ключевые риски и митигация

| Риск                               | Вероятность | Влияние     | Митигация                                                              |
| ---------------------------------- | ----------- | ----------- | ---------------------------------------------------------------------- |
| Блокировка парсеров площадками     | Средняя     | Высокое     | Гибридный retrieval (Tavily + Browser Rendering), HITL при капче       |
| Изменение правил площадок          | Средняя     | Среднее     | Автомониторинг страниц правил, переобучение Agent Memory раз в квартал |
| Галлюцинации LLM                   | Высокая     | Критическое | Только черновики, обязательный HITL, регулярный Eval Suite             |
| Рост стоимости при масштабировании | Средняя     | Среднее     | Per-tenant Cost Tracking, миграция на собственный LLM в v2.1           |
